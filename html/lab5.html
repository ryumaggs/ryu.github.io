Exercise 1

For this exercise you'll be writing changeMaker.py, a program that computes how to give change using the fewest coins.
Your program should:

  • Ask the user for the amount of change
  • Determine how many of each coin you will need
  • Output the results as shown below:

Enter the total amount in cents (< 100): 33
Here's the change that uses the fewest coins:
pennies: 3
nickels: 1
dimes: 0
quarters: 1

Notes and Hints:

  • For each coin you will need to do the following
    • Find the number of coins of that value you'll need (integer division)
      • ex. the number of quarters is the total//25
    • Find the remaining change once you've taken those coins out (modulus)
      • Remember the modulus operator gives you the remainder of a division
      • ex. the change left after taking quarters out is the total%25
  • Don't try to use a for loop, this will only complicate things.
  • Try working out the math first, then the pseudocode, and then write the program.
  • You may assume that the number entered by the user is a positive integer less than 100.
  • US coins have the following values: pennies = 1, nickels = 5, dimes = 10, quarters = 25.



For Loops

Recall that for loops have the following syntax (indentation is important):

for <variable> in <sequence>:
		    
	<statements>

Example: 

i = 1
		
for j in [1,2,3,4,5]:
		
	i = i * j
		
print(i)

What does the program compute? Try changing it so that the user decides at which number the program stops.

  • For this you will need eval(input("...")) and range(...)



Exercise 2

For this exercise you will write power.py, a program that gives the result of one number raised to the power of another number.
Your program should:

  • Ask the user for a base
  • Ask the user for a power
  • Use a for loop to determine the result
  • Output the result
Notes and Hints:
  • You cannot use the ** operator in this exercise
  • You can assume the user inputs integers
  • ex. If the user enters a base of 2 and a power of 3, the result should be 8.
